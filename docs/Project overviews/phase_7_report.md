# Phase 7: Multitasking & Preemption - Technical Report

## 1. Objective
The goal of Phase 7 was to transition the kernel from a single-tasking "shell-only" system to a true multi-core, preemptive operating system capable of running isolated User-Mode (Ring 3) processes.

## 2. Infrastructure Implemented

### Per-CPU Isolation
Each CPU core now maintains its own **GDT** (Global Descriptor Table) and **TSS** (Task State Segment). This is critical because:
- **GS_BASE**: Each core uses the `GS` register to point to its own `PerCpu` struct (containing its kernel stack and private metadata).
- **TSS.rsp0**: Each core must have a private kernel stack to land on when an interrupt occurs while in user mode.

### Round-Robin Scheduler
We implemented a kernel-side scheduler that maintains a global `PROCESS_TABLE`.
- **Preemption**: The APIC Timer is configured to fire periodically. On every tick, the kernel saves the current process context and picks the next `Ready` task.
- **Context Switching**: We use a custom assembly handler (`timer_interrupt_handler_asm`) that saves 15 GPRs and the hardware frame, then swaps the `RSP` to the new process's stack.

## 3. The Debugging Saga (Post-Mortem)

We encountered several "impossible" bugs during this phase. Here is how we solved them:

### Bug A: The Alignment Crash
**Symptoms**: Random Page Faults or GPFs during `iretq`.
**Cause**: The `InterruptFrame` struct was marked `#[repr(C, packed)]`. This caused Rust to shift field offsets by 1-7 bytes, resulting in the CPU reading `0x0` for the `SS` register.
**Fix**: Removed `packed` and enforced `#[repr(C)]` with strict 16-byte alignment.

### Bug B: The Stack Drift
**Symptoms**: System would run for a few seconds, then crash.
**Cause**: The `syscall` handler was saving the `RSP` back into `PerCpu.kernel_stack`. Since the stack pointer changes with every call, the kernel "lost" its stack top over time.
**Fix**: `PerCpu.kernel_stack` now always points to the **Static Top** of the stack.

### Bug C: The SYSRET Phantom (The Final Boss)
**Symptoms**: `CS` selector randomly becoming `0x2b` (Index 5) after a syscall.
**Cause**: The `STAR` MSR was configured with `Base = 0x18`. `SYSRET` calculated `User CS = 0x18 + 16 = 0x28` (Index 5 - TSS).
**Fix**: Changed `STAR` base to `0x10`. Now `User CS = 0x10 + 16 = 0x20` (Index 4 - User Code).

## 4. Inter-Process Communication (IPC)

We successfully established a robust IPC mechanism allowing isolated processes to exchange data:
- **File Descriptor Table**: Extended the `Process` struct to include a dynamic array of File Descriptors (`Vec<Option<FileDescriptor>>`). Standard IO (stdin, stdout, stderr) are pre-mapped to 0, 1, and 2.
- **Ring Buffers (`Pipe`)**: Implemented a thread-safe, mutex-guarded `VecDeque` ring buffer mechanism inside the kernel to serve as a high-speed data channel between processes.
- **Yieldless Blocking State**: Upgraded the scheduler with a `Blocked` state. Processes attempting to read from empty pipes or write to full pipes are put to sleep to save CPU cycles, and are instantly awakened (`Ready`) when the pipe state changes.
- **SYS_SPAWN (Syscall 400)**: Created a modern spawn mechanism that launches new ELF executables from the filesystem and seamlessly clones the parent's File Descriptor table to the child process.
- **Graceful Termination**: Fixed GP faults caused by user-space `hlt` instructions by fully migrating all user-space executables to use the newly refined `SYS_EXIT` syscall.
